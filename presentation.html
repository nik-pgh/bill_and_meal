Bill & Meal 08 / 10
06 — Evaluation 한계
Teacher forcing

Loss가 자기 출력 누적은 평가 안 함

Loss = 매 위치에서 "이전 토큰을 보고 다음 토큰을 얼마나 잘 맞췄나"의 cross-entropy 평균. 낮을수록 좋음. 학습/eval 모두 같은 방식으로 계산.

학습 / eval — Teacher forcing

매 위치에 정답을 깔고 다음 한 토큰만 평가 (한 번의 forward로 전체 시퀀스)

input :  [Mix] [the] [eggs] [and] [add]   ← 모두 정답 토큰
predict: → "salt"   ← 한 토큰만 측정 (cross-entropy)

병렬 처리 가능 → 빠름·안정적인 학습 신호. 그러나 모델 자기 출력이 input으로 들어간 적은 한 번도 없음.

배포 — Autoregressive generate

토큰 하나씩, 자기 출력을 다음 input에 다시 붙이며 진행

step 1: [Mix]            → "the"
step 2: [Mix, the]       → "eggs"
step 3: [Mix, the, eggs] → "and"   ...

모델 자기 출력이 누적적으로 input으로 들어감. 학습에서 한 번도 본 적 없는 입력 분포 (exposure bias). 한 번 어긋나면 뒤로 갈수록 더 어긋남.

v3 evidence

v3 모델은 val_loss 0.59 (정상)인데 generate는 "rice rice rice rice ...". Teacher-forced loss는 매 위치를 정답에 conditioning하므로 누적 collapse를 잡을 신호 자체가 없음. → ingredient_iou / Claude judge 같은 메트릭은 진짜 generate가 필요해서 학습 후 scripts/evaluate.py로 따로 측정.

Bill & Meal — 현재 상태

Pipeline
  • Receipt 이미지 → Claude API → recipe JSON 102개
  • 같은 입력에 같은 출력을 내도록 Gemma 4 E2B 를 QLoRA로 fine-tune
  • 최종 산출물은 LoRA adapter (~수백 MB)
Setup
  • Model: Gemma 4 E2B (T4) / E4B (A100·L4)
  • LoRA: r=16, α=32, language_model only
  • Train: 4-bit NF4, bf16, eff. batch 16, 5 epochs
현재 상태 (미해결)
  • v3 (LoRA 양쪽): repetition collapse — 단, generate 파라미터 영향 가능성 있음
  • v4 (LLM-only LoRA): 학생이 이미지를 인식 못 함
  • val_loss만으론 generation 품질 못 잡음 → epoch별 generation callback 추가
다음
  • Generate ablation — v3에 repetition_penalty만 붙여 재현 여부 확인
  • 데이터 1000+ 확장 후 vision tower LoRA 재시도
  • 중간안: vision tower만 작은 r로, LLM은 freeze
